Linus Torvalds nie jest przeciwnikiem Rust w jądrze Linuxa, ale podchodzi do tematu pragmatycznie. Z jego wypowiedzi wynika, że:  

1. **Rust ożywia dyskusję** – Torvalds ceni argumenty i uważa, że kontrowersje wokół Rust pokazują, jak bardzo ludziom zależy na jakości jądra.  

2. **Opór jest naturalny** – Wielu doświadczonych maintainerów opiera się zmianom, ponieważ Rust wymaga innego podejścia do obsługi błędów niż C, a oni przez dekady pracowali inaczej.  

3. **Rust nie jest zagrożeniem dla Linuxa** – Nawet jeśli integracja Rust się nie powiedzie, będzie to cenna lekcja. Poza tym Linux jest już tak wszechobecny, że alternatywne systemy pisane w Rust nie stanowią realnego zagrożenia.  

4. **Linux nie jest jedynym rozwiązaniem** – Zawsze istniały mniejsze systemy operacyjne dla wbudowanych urządzeń, a ktoś prędzej czy później spróbuje stworzyć nowy system w Rust – ale to nie oznacza, że zastąpi Linuxa.  

Podsumowując: Torvalds nie jest fanatykiem Rust, ale traktuje go jako ciekawy eksperyment. Widzi problemy organizacyjne i techniczne, ale nie uważa ich za przeszkodę nie do pokonania.
