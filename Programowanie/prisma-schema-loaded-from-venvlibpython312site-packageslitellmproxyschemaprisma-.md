# Prisma schema loaded from venv/lib/python3.12/site-packages/litellm/proxy/schema

**Data:** 23 gru 2025, 19:44:05 CET

---

**User:**

Prisma schema loaded from venv/lib/python3.12/site-packages/litellm/proxy/schema.prisma
Error: Generator "prisma-client-py" failed:

/bin/sh: 1: prisma-client-py: not found

---

**Gemini:**

Błąd `/bin/sh: 1: prisma-client-py: not found` oznacza, że binarny CLI Prismy (napisany w Rust/Node) próbuje wywołać generator Pythona, ale nie może go odnaleźć w zmiennej środowiskowej `PATH` powłoki systemowej. Mimo że wywołujesz `prisma` z pełnej ścieżki, proces potomny (sub-process), który generuje klienta, nie dziedziczy automatycznie ścieżek do binariów Twojego środowiska wirtualnego.

Aby to naprawić, musisz jawnie dodać katalog `bin` Twojego `venv` do zmiennej `PATH` podczas wykonywania komendy.

### 1. Generowanie z jawnym eksportem PATH

Użyj poniższej komendy. Kluczowe jest dodanie `/opt/litellm-proxy/venv/bin` na początku `PATH`:

```
sudo -u litellm bash -c "
export PATH='/opt/litellm-proxy/venv/bin:\$PATH' && \
export DATABASE_URL='postgresql://litellm_user:your_secure_password@localhost:5432/litellm_db' && \
export HOME='/home/litellm' && \
export XDG_CACHE_HOME='/home/litellm/.cache' && \
/opt/litellm-proxy/venv/bin/prisma generate --schema /opt/litellm-proxy/venv/lib/python3.12/site-packages/litellm/proxy/schema.prisma
"
```

*(Zauważ znak `\` przed `$PATH` – zapobiega to przedwczesnej interpolacji zmiennej przez Twoją obecną powłokę, zanim `sudo` ją wykona).*

### 2. Dlaczego to jest konieczne?

Wewnątrz pliku `schema.prisma` znajduje się blok:

```
generator client {
  provider = "prisma-client-py"
}
```

Kiedy uruchamiasz `prisma generate`, binarka CLI szuka w systemie pliku wykonywalnego o nazwie `prisma-client-py`. Ponieważ środowisko wirtualne nie jest "aktywowane" w tradycyjny sposób (`source activate`), systemowa powłoka `/bin/sh` nie wie, że ten plik znajduje się w `/opt/litellm-proxy/venv/bin/`.

### 3. Weryfikacja obecności generatora

Upewnij się, że plik generatora faktycznie istnieje w Twoim `venv`:

```
ls -l /opt/litellm-proxy/venv/bin/prisma-client-py
```

Jeśli pliku nie ma, zainstaluj go jawnie:

```
sudo -u litellm /opt/litellm-proxy/venv/bin/pip install prisma-client-py
```

### 4. Alternatywne rozwiązanie (Symbolic Link)

Jeśli powyższy eksport `PATH` z jakiegoś powodu by zawiódł, możesz tymczasowo stworzyć link symboliczny w katalogu, który na pewno jest w `PATH` (np. `/usr/bin`), choć metoda z `PATH` wewnątrz `bash -c` jest czystsza:

```
sudo ln -s /opt/litellm-proxy/venv/bin/prisma-client-py /usr/bin/prisma-client-py
```

### 5. Kolejny krok po sukcesie `generate`

Gdy komenda `generate` zakończy się sukcesem, wykonaj `db push`, aby zsynchronizować tabele z PostgreSQL (pamiętaj o dodaniu `PATH` również tutaj):

```
sudo -u litellm bash -c "
export PATH='/opt/litellm-proxy/venv/bin:\$PATH' && \
export DATABASE_URL='postgresql://litellm_user:your_secure_password@localhost:5432/litellm_db' && \
export HOME='/home/litellm' && \
/opt/litellm-proxy/venv/bin/prisma db push --schema /opt/litellm-proxy/venv/lib/python3.12/site-packages/litellm/proxy/schema.prisma
"
```

Po tych operacjach zrestartuj serwis:

```
sudo systemctl restart litellm
```

Czy po dodaniu `PATH` do komendy generator uruchomił się poprawnie i otrzymałeś komunikat o wygenerowaniu klienta Pythona?
